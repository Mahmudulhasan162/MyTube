Question: Discuss the scope of var, let, and const

Answer:
To declare variables in JavaScript we use var, let and const. But each of them has different scoping rules.

var: Variables that are declared using var are either function scoped or global scoped but not blocked scoped. That means the variables declared by var in a function can be used anywhere inside the function and same as global. var is not blocked scoped, that means if we declare a var in a blocked scoped that can be accessible from outside the scope.
let: Variables declared with let is blocked scoped that means these variables can only be accessible inside the block. It can not be accessed outside it, but we can reassigned the variable value within the block.
const: Variables declared with const are also blocked scoped but they have something unique which is we can't reassigned the value after they are initially assigned. That makes the variable to have a constant value.

Question: Tell us the use cases of null and undefined

Answer: In JavaScript, when there are lack of defined values or absence of a value found, then JavaScript automatically provide two values which are 'null' and 'undefined'.

null:
1.Explicit Absence of Value: When JavaScript found there is explicit assignment of a variable as 'null' then it returns 'null' as a value.
2. Object property value: If a property of an object in JavaScript has 'null' value, that indicates the property exists but value doesn't exist.

undefined:
1. Default Value: When JavaScript found there is no assignment of a variable then it returns 'undefined' as a value.
2. Parameter of Function: When JavaScript found there is no value assigned as parameter of a function when it is called then it returns 'undefined' as a default.
3. Property Access: When we try to access a property of an object which is not defined then JavaScript return 'undefined'

Question: What do you mean by REST API?

Answer: REST API (Representational State Transfer Application Programming Interface) is a set of architectural principles and conventions for designing website services. It allows different software applications to communicate over the internet using the HTTP methods. REST APIs are resources represented by URLs and are characterized, and the use of HTTP methods for operations on these resources, making them widely used for building scalable web services.
